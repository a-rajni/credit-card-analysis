# credit-card-analysis
I built a Credit Card Analytics dashboard in Power BI to analyze customer behavior, revenue, and transaction patterns. The report provides insights into customer demographics, spending habits, and card usage. It helps stakeholders understand which customer segments generate the most revenue and how transactions vary by category, card type, and time.


**1. Data & Preparation**

Imported customer and credit card transaction data
Cleaned data using Power Query (handled nulls, formatted columns, created categories like age group, income level)
Built relationships between tables (customer & transaction)

**2. Dashboard 1: Customer Report**

Shows customer segmentation
By occupation (businessman, white-collar, etc.)
By education level
By age group
By income group

Key KPIs:
Total Revenue
Transaction Amount
Total Interest Earned
Average Customer Rating

Insights:
Which customer group generates more revenue
Which age/income segment is most valuable

**3. Dashboard 2: Transaction Report**
Focuses on spending behavior
Revenue split by:
Expenditure type (bills, fuel, grocery, etc.)
Card type (silver, gold, platinum)
Usage type (swipe, chip, online)
Time analysis:
Quarterly transaction trends

Key KPIs:
Total transactions count
Total revenue
Interest earned

**4. Filters**

Slicers for:
Gender (Male/Female)
Quarter (Q1–Q4)
Card category
Week number
Enables dynamic analysis for business users

**This dashboard helps the business:**
Identify high-value customers
Understand spending patterns
Optimize marketing strategies
Improve credit card offerings based on usage trends

I found that swipe transactions contribute the highest revenue, and customers aged 40–60 generate the most income. Also, certain occupations like businessmen contribute significantly more to total revenue.
